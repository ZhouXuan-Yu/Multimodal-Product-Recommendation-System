class VectorDBManager:
    def __init__(self, persist_path, collection_name):
        self.client = chromadb.PersistentClient(path=persist_path)
        self.collection = self.client.get_collection(name=collection_name)
        self.device = "cuda" if torch.cuda.is_available() else "cpu"
        # 加载特征提取模型 (与入库时必须一致)
        self.model, self.preprocess = load_clip_model(self.device)

    async def search(self, text=None, image_path=None, top_k=10, price_range=None):
        # 1. 生成查询向量
        query_vector = self._generate_embedding(text, image_path)
        
        # 2. 构造过滤器 (Commercial Level Filtering)
        where_clause = {}
        if price_range:
            where_clause["price"] = {"$lte": price_range[1], "$gte": price_range[0]}

        # 3. 执行向量检索
        results = self.collection.query(
            query_embeddings=[query_vector],
            n_results=top_k,
            where=where_clause,
            include=["metadatas", "distances"]
        )
        
        # 4. 格式化并返回
        return self._format_results(results)

> - **负反馈处理：** 如果用户点击“不喜欢”，则在下次 Rerank 的 Prompt 中加入“排除类似风格/品牌”的强约束。

## 一、 系统性能压测与并发优化：压榨 RTX 5060 的极限

商业系统要求低延迟。你的瓶颈通常不在 DeepSeek API（那是云端），而是在**本地 Embedding 推理**和 **向量库 I/O**。

## 1. 核心优化策略

- **模型量化与半精度 (FP16)：**
    
    - 在 RTX 5060 上运行 CLIP 或 BGE 模型时，务必使用 `half()` 精度。这能节省 **50% 的显存**，并提升近 **2 倍** 的推理速度。
        
- **向量检索分片与索引优化：**
    
    - ChromaDB 在数据量大时，查询速度会下降。确保开启了 **HNSW 索引**。
        
    - **缓存机制：** 使用 Redis 缓存高频 Query 的向量。如果用户搜过“黑色移动硬盘”，10 分钟内再次搜索，直接读缓存，不经过 GPU。
        
- **异步并发流水线 (Asynchronous Pipeline)：**
    
    - 不要串行：`意图解析 -> 向量提取 -> 检索 -> Rerank`。
        
    - 要并行：在 DeepSeek 解析意图的同时，预加载用户画像数据；在检索数据的同时，异步记录用户行为日志。
        

## 2. 压测方案 (Stress Testing)

- **工具推荐：Locust**（Python 编写，非常适合 FastAPI 项目）。
    
- **关键指标：**
    
    - **RT (Response Time)：** 理想情况下，从输入到看到推荐结果应控制在 **800ms** 以内（含 API 往返）。
        
    - **QPS (Queries Per Second)：** 测试你的 5060 在显存溢出前能同时处理多少个并发 Embedding 请求。

## 🎨 优化提示词一：基础图表 (Basic Charts) —— 从“统计”到“画像”

**现状分析：** 目前页面 主要是传统的点击计数和品类饼图。

**优化目标：** 展示“视觉 DNA”和“多模态贡献度”。

> **提示词 (Phase 1):** > 1. **组件重构：** 将现有的“类目偏好分布”饼图升级为 **“视觉审美雷达图”**。
> 
> - **维度：** 极简、技术感、商务、复古、机能。
>     
> - **数据逻辑：** 从 ChromaDB 的图像特征向量中聚合提取。
>     
> 
> 2. **趋势分析优化：** 在“趋势分析”折线图中加入 **“意图波峰”**。
>     
>     - **交互：** 鼠标悬停在波峰处，弹出 DeepSeek 生成的语义摘要（如：“该时段用户正在深度调研 RTX 5060 相关配件”）。
>         
> 3. **新增组件：** 增加一个“数据贡献环形图”，展示推荐来源占比：$30\%$ 文本搜索、$50\%$ 图像浏览记录、$20\%$ 历史交互。
>

## 🔍 优化提示词二：数据钻取 (Data Drill-down) —— 追踪“意图偏移”

**现状分析：** 目前主要基于标签下钻。

**优化目标：** 可视化用户偏好是如何随时间“漂移”的。

> **提示词 (Phase 2):**
> 
> 1. **交互逻辑：** 实现从“类目 -> 具体商品 -> 向量影响值”的深度下钻。
>     
> 2. **核心可视化：** 开发一个 **“向量偏移轨迹图”**。
>     
>     - **内容：** 展示该商品点击后，用户全局画像向量在 2D 降维空间中的位移方向。
>         
>     - **语义表达：** 自动标注该行为让画像“更偏向高性能数码”或“转向生活品质”。
>         
> 3. **行为轨迹流：** 采用桑基图 (Sankey Diagram) 展示用户从“搜索关键词”到“查看详情”再到“最终反馈”的多模态路径。
>

## 📊 优化提示词三：数据对比 (Data Comparison) —— 语义化差异分析

**现状分析：** 现有页面 展示了本期与上期的数值对比。

**优化目标：** 让 DeepSeek 解释“为什么会有变化”。

> **提示词 (Phase 3):**
> 
> 1. **对比逻辑优化：** 除了数值百分比，增加 **“语义差分看板”**。
>     
>     - **逻辑：** 比较两个时间段内 Top 10 商品的平均向量重心。
>         
> 2. **AI 叙事组件：** 由 DeepSeek 生成对比总结。
>     
>     - **示例：** “本期您的兴趣焦点从‘通用数码’明显迁移至‘专业 AI 硬件’，意图强度提升了 $15\%$。”
>         
> 3. **视觉反馈：** 使用红/绿动态呼吸灯标识关键指标的“健康度”，而非简单的数字颜色。
>

## 🚨 优化提示词四：异常检测 (Anomaly Detection) —— 捕捉“新兴兴趣”

**现状分析：** 目前使用 Z-Score 检测事件量异常。

**优化目标：** 识别“兴趣突变”或“风格偏移”。

> **提示词 (Phase 4):**
> 
> 1. **算法可视化：** 在折线图上高亮显示超越置信区间 $Z > 2.2$ 的异常点。
>     
> 2. **异常分类：** 区分“突发性强意图”（如急需购买某物）和“新兴趣发现”（如突然开始关注火灾烟雾检测项目）。
>     
> 3. **预测预警：** 结合线性回归，预测如果该异常行为持续，用户的画像向量将在 3 天内发生根本性偏移，并弹出提示：“系统发现您有新的兴趣领域，是否需要调整推荐策略？”
>
