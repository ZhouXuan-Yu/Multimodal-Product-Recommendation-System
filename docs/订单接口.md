# 订单相关 API 对接说明（FastAPI × Vue 前端 × Postman）

本文档说明当前项目中与“下单 / 支付 / 查询”相关的接口设计，便于你在本地通过 **Postman** 或前端 `module_frontend_vue` 直接对接和联调。

后端实现位于：`module_backend/main_api.py`。

## 1. 公共约定

- **基础 URL（本地开发）**：`http://127.0.0.1:8000`
- **前端开发 URL**：`http://127.0.0.1:5173`
- **开发环境代理**：前端通过 Vite 代理 `/api` 到 FastAPI，因此前端只需要请求 `/api/...` 即可：
  - `vite.config.js` 中已经配置：
    - `/api` → `http://127.0.0.1:8000`
    - `/static` → `http://127.0.0.1:8000`
- **商品主数据**：`data/meta/products.json`，字段中 `product_id` 会被用于订单里的每一项。
- **订单持久化文件**：`data/meta/orders.json`，结构为：

```json
{
  "orders": {
    "local-user_001-20250301010101000": {
      "order_id": "local-user_001-20250301010101000",
      "user_id": "user_001",
      "status": "pending",
      "items": [
        {
          "product_id": "p001",
          "quantity": 2,
          "unit_price": 199.0
        }
      ],
      "total_amount": 398.0,
      "currency": "CNY",
      "created_at": "...",
      "updated_at": "...",
      "note": "用户备注",
      "payment_channel": "mock_pay",
      "paid_at": "..."
    }
  }
}
```

前端会调用的订单相关接口有 5 个：

- `POST /api/orders/preview`：订单预览（不落盘）
- `POST /api/orders`：创建订单（落盘）
- `POST /api/orders/pay`：模拟支付
- `GET /api/orders/{order_id}`：获取单个订单详情
- `GET /api/orders`：订单列表

下面分别说明。

---

## 2. 订单预览：`POST /api/orders/preview`

**用途**：前端在“结算页”点击“提交订单”前，先计算总价、生成订单摘要，让用户确认。

- **是否写文件**：否（纯内存，`order_id` 为 `"preview"`）
- **典型场景**：`CartCheckout.vue` 中点击“提交订单”先走预览接口。

### 2.1 请求参数（JSON Body）

```json
{
  "user_id": "user_001",
  "currency": "CNY",
  "note": "请尽量当天发货",
  "items": [
    { "product_id": "p001", "quantity": 2 },
    { "product_id": "p002", "quantity": 1 }
  ]
}
```

字段说明：

- **user_id**：用户 ID，字符串。
- **currency**：币种，默认 `"CNY"`。
- **note**：用户备注，可选。
- **items**：订单项列表：
  - `product_id`：必须是 `products.json` 中存在的 `product_id`。
  - `quantity`：数量，≥ 1。

### 2.2 响应示例（200）

```json
{

    "order_id": "preview",
    "user_id": "user_001",
    "status": "pending",
    "items": [

        "product_id": "p002",

```

前端可以直接使用：

- `order.total_amount`：展示合计金额；
- `order.items[*].image_url`：商品图片；
- `summary`：展示在结算确认区域。

### 2.3 Postman 操作提示

- 方法：`POST`
- URL：`http://127.0.0.1:8000/api/orders/preview`
- Body：`raw` + `JSON`，粘贴上方示例。

## 3. 创建订单：`POST /api/orders`

**用途**：确认预览无误后，真正创建订单并写入本地 `orders.json`。

- **是否写文件**：是（生成唯一 `order_id`）。
- **ID 规则**：`local-<safe_user_id>-<timestamp>`，例如：
  - `local-user_001-20250301123456789`

### 3.1 请求参数（JSON Body）

与预览接口完全相同：

```json
{
```

**典型调试流程：**

1. 先调用 `/api/orders/preview` 看计算是否正确；
2. 再调用 `/api/orders` 创建订单；
3. 使用 `/api/orders/{order_id}` 或 `/api/orders` 检查是否已写入。

## 4. 模拟支付：`POST /api/orders/pay`

**用途**：本地环境下不对接真实支付渠道，只是将订单状态从 `"pending"` 改为 `"paid"`，并记录支付渠道与时间。

### 4.1 请求参数（JSON Body）

```json
{
  "order_id": "local-user_001-20250301123456789",
  "payment_channel": "mock_pay"
}
```

字段说明：

- **order_id**：要支付的订单 ID（从创建订单的返回中获取）。
- **payment_channel**：支付渠道标记，当前支持示例值：
  - `"mock_pay"`：本地模拟
  - `"alipay"`：支付宝（未来扩展）
  - `"wechat"`：微信支付（未来扩展）

### 4.2 响应示例（200）

```json
{
  "order": {
    "order_id": "local-user_001-20250301123456789",
    "user_id": "user_001",
    "status": "paid",
    "items": [
      {
        "product_id": "p001",
        "quantity": 2,
        "unit_price": 199.0,
        "name": "多功能复古数码相机",
        "price": 199.0,
        "category": "数码影音",
        "image_url": "/static/images/p001.jpg"
      }
    ],
    "total_amount": 398.0,
    "currency": "CNY",
    "created_at": "2025-03-01T12:35:01.123456",
    "updated_at": "2025-03-01T12:36:10.000000",
    "note": "请尽量当天发货",
    "payment_channel": "mock_pay",
    "paid_at": "2025-03-01T12:36:10.000000"
  }
}
```

### 4.3 常见错误

- **404**：`订单不存在`
  - 可能是 `order_id` 写错，或还未调用创建接口。

## 6. 订单列表：`GET /api/orders`

**用途**：前端展示“我的订单列表”，支持按用户过滤与数量限制。

### 6.1 查询参数

- `user_id`（可选）：只看某个用户的订单。
- `limit`（可选）：返回条数上限，默认 50，最大 200。

请求示例：

```text
GET http://127.0.0.1:8000/api/orders?user_id=user_001&limit=20
```

### 6.2 响应示例（200）

```json
{
  "orders": [
    {
      "order_id": "local-user_001-20250301123456789",
      "user_id": "user_001",
      "status": "paid",

        {
          "product_id": "p001",
          "quantity": 2,

          "price": 199.0,

          "image_url": "/static/images/p001.jpg"
        }
      ],
      "total_amount": 398.0,
      "currency": "CNY",
      "created_at": "2025-03-01T12:35:01.123456",
```

## 7. 与前端 `module_frontend_vue` 的对接建议

后续前端会在 `src/api.js` 中补充以下封装函数，统一与上述接口对齐：

- `previewOrder(payload)` → `POST /api/orders/preview`
- `createOrder(payload)` → `POST /api/orders`
- `payOrder(payload)` → `POST /api/orders/pay`
- `fetchOrders(params)` → `GET /api/orders`

推荐的结算流程（对应前端组件 `CartCheckout.vue`）：

1. 用户在商城中通过 `CartModal` 加入商品到购物车；
2. 在 `CartCheckout.vue` 点击“提交订单”：
   - 先调用 `previewOrder` 获取 `order` + `summary`，展示确认弹窗或信息区；
   - 用户点击“确认下单”后，再调用 `createOrder`，拿到真实 `order_id`；
3. 可选：自动调用 `payOrder` 将状态置为 `paid`，或提供“去支付”按钮；
4. 前端可将成功的订单追加到“我的订单列表”视图中，或跳转到“订单详情”视图。

在你阅读和调整前端代码时，可以随时打开本文件，对照接口字段进行调试与扩展。若实际返回结构有改动，建议同步更新这里的示例。
