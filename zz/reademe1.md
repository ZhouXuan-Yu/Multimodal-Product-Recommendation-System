# 毕业设计：基于多模态融合的商品个性化推荐系统

## 技术开发说明书 (Cursor 专用版)

### 1. 项目背景与目标

本系统旨在通过融合商品图像（视觉特征）与文本描述/评论（语义特征），解决传统推荐系统中数据稀疏和兴趣挖掘不足的问题 。系统需涵盖从多模态数据采集、特征提取与融合，到推荐结果生成与可视化反馈的闭环流程 。

### 2. 技术栈架构

* **语言与框架**: Python 3.10+, PyTorch (深度学习核心)。
* **多模态编码器**:
* 
**文本 (NLP)**: BERT (提取 768 维语义向量，取代传统的 Word2Vec )。


* 
**图像 (CV)**: ResNet-50 (提取 2048 维视觉向量，取代 AlexNet )。




* 
**数据库**: MySQL 8.0 (管理用户行为及商品元数据 )。


* 
**GUI 界面**: PyQt6 (实现美观、响应式的电商瀑布流布局 )。



---

### 3. 数据库物理设计 (MySQL)

设计采用路径存储和复合索引，以确保大量图片与文本的高效检索。

* **商品表 (`products`)**: 存储 `product_id` (PK), `title`, `img_path` (本地磁盘路径), `description`。
* **用户表 (`users`)**: 存储 `user_id` (PK), `username`, `pref_tags` (用户偏好标签)。
* **行为表 (`user_actions`)**: 存储 `action_id`, `user_id` (FK), `product_id` (FK), `action_type` (view/buy), `timestamp`。
* **推荐表 (`recommendations`)**: 存储 `user_id` (FK) 对应的 `rec_list_json` (推荐结果缓存)。

### 5. PyQt6 界面交互逻辑

界面设计需符合 2026 年电商审美，采用**异步加载**防止推理过程导致 UI 卡死。

* **布局逻辑**: 顶部搜索栏 + 左侧导航栏 + 中央多模态商品瀑布流。
* **多线程通信**: 使用 `QThread` 封装 PyTorch 推理逻辑，通过信号 (Signal) 传递推荐结果。
* **交互细节**: 用户点击商品实时更新 `user_actions` 表，并触发兴趣向量更新。

### 6. 实验与评估方案

通过**消融实验 (Ablation Study)** 证明多模态的价值 。

| 实验组 | 描述 | 关键指标 |
| --- | --- | --- |
| **Baseline** | 传统协同过滤 (CF) | Precision@K |
| **Text-Only** | 仅基于 BERT 文本特征 | Recall@K |
| **Image-Only** | 仅基于 ResNet 图像特征 | F1-Score |
| **Full-Fusion** | 本文设计的多模态融合系统 | CTR, MRR |
